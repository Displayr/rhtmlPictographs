It is possible to specify the size of an image slot in a single image or multi image graphic cell. The cell will then dynamically size to support the specified image width or image height.

In order to create graphic cells with fixed size image slots, two additions to the config are necessary:

  • The rowHeights and colWidths must be set to "fixedsize:graphic"
  • The imageWidth, and/or the imageHeight must be specified in the graphic cell definition
Because you must specify the rowHeights/colWidths as "fixedsize:graphic", it is not possible to use the graphic cell shorthand; you must explicitly pass in a table.rows field containing the matrix of cells that you wish to render.

Specifying both imageWidth and imageHeight to a fixed size graphic cell will force each image to match the specified size

When specifying just imageWidth, imageHeight will be computed based on aspect ratio of image. For shapes it will assume aspect ratio of 1

When specifying just imageHeight, imageWidth will be computed based on aspect ratio of image. For shapes it will assume aspect ratio of 1

Graphic cells with fixed image slots play well with both table footers and graphic cell text footers

{
      "table-footer": { "text": "table footer", "font-size": "14px" },
      "table": {
        "colWidths": [ "fixedsize:graphic" ],
        "rowHeights": [ "fixedsize:graphic" ],
        "rows": [
          [
            {
              "type": "graphic",
              "value": {
                "proportion": "=4/6",
                "numImages": 6,
                "numRows": 2,
                "imageWidth": 10,
                "imageHeight": 20,
                "rowGutter": 0.2,
                "columnGutter": 0.2,
                "variableImage": "rect:fromtop:lightblue",
                "baseImage": "rect:#CCCCCC"
              }
            }
          ]
        ]
      }
    }
